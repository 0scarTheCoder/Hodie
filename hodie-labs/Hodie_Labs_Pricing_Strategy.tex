\begin{itemize}[leftmargin=*]

\subsubsection{Free Tier Strategy - Zero AI Cost Solution}

\textbf{Critical Issue:} Claude AI has no free tier. Every API call costs money.

\textbf{Solution - Use Groq (Free Tier):}
\begin{itemize}
    \item Groq offers \textbf{free API access} to Llama 3 8B (14,400 requests/day)
    \item Fast inference (300+ tokens/second)
    \item Quality: 70\% as good as Claude Haiku
    \item Cost: \textbf{\$0.00} for 10 messages/month
\end{itemize}

\textbf{Alternative Options:}
\begin{enumerate}
    \item \textbf{Pre-computed Responses:} FAQs with canned answers (\$0 cost)
    \item \textbf{Claude Haiku (Minimal):} Reduce to 5 messages/month (\$0.04/user)
    \item \textbf{Kimi K2:} Chinese LLM with cheaper rates (\$0.02/user)
\end{enumerate}

\textbf{Recommended:} Use Groq Llama 3 for free tier = \textbf{\$0 AI cost}.

\section{DigitalOcean Infrastructure Costs}

\subsection{Why DigitalOcean?}

Hodie Labs will use \textbf{DigitalOcean} as the primary cloud computing platform for:

\begin{itemize}[leftmargin=*]
    \item \textbf{Managed MongoDB} - Database hosting for user data

\section{Critical Implementation Details}

\subsection{Free Tier - Technical Implementation}

\textbf{Current Problem:} Your Claude API key is exposed in browser (security risk).

\textbf{Solution - Backend API Proxy:}

\begin{enumerate}
    \item Build Node.js/Express backend
    \item Endpoint: \texttt{POST /api/chat}
    \item Backend checks user tier, selects appropriate model:
    \begin{itemize}
        \item Free → Groq Llama 3 (free API)
        \item Basic/Pro → Claude Haiku
        \item Premium → Claude 3.5 Sonnet
    \end{itemize}
    \item Backend enforces rate limits (10/100/500/1000 messages)
    \item API keys stored securely on backend only
\end{enumerate}

\textbf{Cost Savings:}
\begin{itemize}
    \item Free tier: \$0.23/user → \$0.08/user (65\% reduction)
    \item Prevents API key theft/abuse
    \item Enables per-user rate limiting
\end{itemize}

\subsection{Data Efficiency - Summarisation Strategy}

\textbf{Current Problem:} Sending 20 sample rows (2,000 tokens) in every message.

\textbf{Solution - Pre-computed Summaries:}

\begin{enumerate}
    \item On file upload, compute statistics:
    \begin{itemize}
        \item Mean, median, std dev for all numeric fields
        \item Distribution histograms
        \item Trend analysis (increasing/decreasing)
        \item Anomaly detection
    \end{itemize}
    \item Store summary in MongoDB (200 tokens vs. 2,000)
    \item Send summary to Claude, not raw data
    \item Only fetch raw data when user specifically requests it
\end{enumerate}

\textbf{Cost Savings:}
\begin{itemize}
    \item 90\% reduction in tokens per message
    \item Basic tier: \$1.87 → \$0.40/user
    \item Pro tier: \$5.90 → \$1.30/user
\end{itemize}

\subsection{Prompt Caching Implementation}

Claude API supports prompt caching (reduces costs by 90\% for repeated content).

\textbf{How it works:}
\begin{itemize}
    \item Cache system prompt + user data summary
    \item First message: Full cost
    \item Next messages (within 5 minutes): 90\% discount on cached portion
    \item Typical conversation: 5-10 messages in 5 minutes
\end{itemize}

\textbf{Cost Reduction Example:}
\begin{itemize}
    \item User has 10-message conversation
    \item Without caching: 10 × 2,000 tokens = 20,000 tokens
    \item With caching: 2,000 + (9 × 200) = 3,800 tokens (81\% reduction)
\end{itemize}

\textbf{Expected Savings:} 50-70\% on active users (those with multi-message conversations).

\subsection{Rate Limiting \& Fair Use Policy}

\textbf{Implementation:}
\begin{itemize}
    \item Store message count per user per month in MongoDB
    \item Backend checks count before processing
    \item Return friendly error: ``You've reached your monthly limit. Upgrade to Pro for 500 messages/month.''
\end{itemize}

\textbf{Fair Use Policy (Premium ``Unlimited''):}
\begin{itemize}
    \item Soft cap: 1,000 messages/month
    \item If user exceeds 1,500 messages: Review account
    \item 99\% of users will stay under 1,000
    \item Prevents abuse while keeping ``unlimited'' promise
\end{itemize}

\section{Key Metrics to Track}

\subsection{Financial Metrics}

\begin{enumerate}[leftmargin=*]
    \item \textbf{MRR} (Monthly Recurring Revenue): Total subscription revenue
    \item \textbf{ARR} (Annual Recurring Revenue): MRR × 12
    \item \textbf{ARPU} (Average Revenue Per User): Total revenue ÷ paid users
    \item \textbf{CAC} (Customer Acquisition Cost): Marketing spend ÷ new users
    \item \textbf{LTV} (Lifetime Value): ARPU × average subscription months
    \item \textbf{LTV:CAC Ratio}: Target 3:1 or higher
    \item \textbf{Gross Margin}: (Revenue - COGS) ÷ Revenue
    \item \textbf{AI Cost Per User}: Track per tier monthly
\end{enumerate}

\subsection{User Metrics}

\begin{enumerate}[leftmargin=*]
    \item \textbf{Free-to-Paid Conversion Rate}: Target 20\%+
    \item \textbf{Churn Rate}: Monthly cancellations ÷ total users (target <5\%)
    \item \textbf{Upgrade Rate}: Users moving from Basic → Pro → Premium
    \item \textbf{Messages Per User}: Track usage patterns
    \item \textbf{File Uploads Per User}: Engagement metric
    \item \textbf{Daily Active Users (DAU)}: Active engagement
    \item \textbf{Monthly Active Users (MAU)}: Overall platform health
\end{enumerate}

\subsection{AI Performance Metrics}

\begin{enumerate}[leftmargin=*]
    \item \textbf{Average Tokens Per Message}: Optimise to reduce costs
    \item \textbf{Cache Hit Rate}: \% of messages using cached prompts
    \item \textbf{Model Split}: \% using Haiku vs. Sonnet
    \item \textbf{Error Rate}: Failed API calls / total calls
    \item \textbf{Response Time}: P95 latency (target <3 seconds)
\end{enumerate}
